03//The four collections
One table, four mental models. Pick the right tool for the job.
TypeSyntaxOrdered? Mutable?Duplicates?Use it for…
list [1, 2, 3] yes yes yes ordered sequences you'll modify
tuple (1, 2, 3) yes no yes fixed records, function returns
dict {"k": v} yes yes keys: no key → value lookups (O(1))
set {1, 2, 3} no yes no uniqueness & membership tests
# ordered in Python 3.7+; in dicts that means insertion order is preserved

When to reach for a list
  • order matters
  • you'll add, remove, or sort items
  • duplicates are fine
  • you index by position, not name
Common methods

Why dicts are fast
Lookup by key is O(1) — the same speed whether you have 10 items or 10 million.

That's why every blockchain RPC returns dicts.

07//Sets — unique items, fast membership tests
sets_demo.py
# every sender who appeared in block #4521
senders = ["alice", "bob", "alice", "eve"]

unique = set(senders)    # {'alice', 'bob', 'eve'}

# blazing-fast membership test — O(1)
"bob" in unique           # True

# set algebra works as math
block_a = {"alice", "bob"}
block_b = {"bob", "eve"}
block_a | block_b         # union
block_a & block_b         # intersection → {'bob'}
Reach for a set when…
  • you need to dedupe a list
  • you check membership often
  • you do union / intersection
  • order genuinely doesn't matter
On-chain use
Daily Active Addresses is just len(set(senders)) across a day's blocks.

08//List comprehensions — the Pythonic for-loop
SYNTAX
[ expression  for item in iterable  if condition ]
# the if-clause is optional
Before · the long way
loop.py
amounts = [12, 200, 8, 450, 75]
big = []
for a in amounts:
    if a > 100:
        big.append(a)

# 5 lines, mutable accumulator
After · the comprehension
comp.py
amounts = [12, 200, 8, 450, 75]

big = [a for a in amounts if a > 100]

# 1 line, no accumulator
# → [200, 450]

# transform too — fees in paise
paise = [a * 100 for a in amounts]

M1·DAY 03·Dict Comprehensions
09/14
09//Dict comprehensions — build maps in one expression
SYNTAX
{ key_expr : value_expr  for item in iterable }
# also works with if-clauses
dict_comp.py
# list of tickers we follow
tickers = ["BTC", "ETH", "DOT", "SOL"]
prices  = [67000, 3400, 7.2, 145]

# zip + dict comprehension → ticker -> price
book = {t: p for t, p in zip(tickers, prices)}
# {'BTC': 67000, 'ETH': 3400, 'DOT': 7.2, 'SOL': 145}

# only the large-caps (price > 100)
large = {t: p for t, p in book.items() if p > 100}

11//Reading & writing files — safely, with `with`
Text files
files_text.py
# write a log line
with open("chain.log", "w") as f:
    f.write("block #4521 ok\n")

# read it back, line by line
with open("chain.log") as f:
    for line in f:
        print(line.rstrip())

# 'with' auto-closes the file —
# even if an exception is raised.
JSON files

         "tx_count": 12}

# read JSON

# we'll write all three live, then defaultdict-up #03
01 · Total volume
total = sum(t["amount"] for t in transactions)    # → 785
02 · Whales only (amount > 100)
whales = [t for t in transactions if t["amount"] > 100]
03 · Sender ledger { sender: total_sent }

    s: sum(t["amount"] for t in transactions if t["sender"] == s)

}
